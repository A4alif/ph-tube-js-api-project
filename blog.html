The Scope of var, let and const

Var

-- The scope of variables declared with the var keyword is function-scoped, which implies that if they are not inside a function, their scope is restricted to the global scope.

-- Variable declarations are elevated to the top of their containing function or global scope

-- Re-declaring a variable with var in the same scope has the potential to produce unexpected results.

Let

-- Variables declared with let are block-scoped,

-- You cannot re-declare a variable with let in the same scope

Const

-- Const variables are likewise block-scoped, much like let variables are.

-- Variables declared with const cannot be reassigned to a new value after they are initially assigned.

-- you cannot re-declare a variable with const in the same scope

Use Cases of null and undefined

Null

-- null is oftern used to represent the intentional absence of any object

-- functions sometimes return null to indicate they didn't find a meaningful result

-- you can set a variable to null to clear it's value

Undefined

-- Variable declared but value isn't assigned then it typically show undefined

-- Access object property that doesn't exit

-- If function doesn't return any value then it returns undefined

What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building and interacting with web services.It actions are performed using standard HTTP methods. REST was first introduced by Roy Fielding in 2000.

HTTP methods

GET - This is used to provide a read only access to a resource.

PUT - This is used to create a new resource.

DELETE - This is used to remove a resource.

POST - This is used to update a existing resource or create a new resource.
